## 2. Data model — grid + assignment

`grid = { [branchId]: Day[7] }` เก็บใน `sched_grid(week_start pk, grid jsonb)`
`Day = { morning:[], mid:[], lunch:[], dinner:[], night:[] }` · index วัน 0=จันทร์ (ISO)

แต่ละช่อง = **assignment**:

```js
{
  id,          // employee uuid
  pos,         // 'cook' | 'kasst' | 'cash'
  via,         // 'home' | 'loan'  (ยืมข้ามสาขา)
  homeBranch,  // สาขาต้นสังกัด (แสดง "ช่วยจาก X")
  locked,      // true = แก้มือ/ประวัติจริง — regen ไม่ย้าย
  t?,          // เวลาจริง เช่น '08:00-20:00' — แสดง + คิด ชม. แทนเวลากะมาตรฐาน
  ext?,        // true = กะยาว 12 ชม. (OT +2) — "บรีสโมเดล"
  off?,        // true = กะช่วยวันหยุด: แสดง+คิดเงิน แต่ไม่นับเป็นวันทำงาน (ยังได้หยุด)
  name?,       // ชื่อคนลาออกแล้ว (ไม่อยู่ใน roster) — ประวัติจริง แสดง "ออกแล้ว"
}
```

> ⚠️ field `t / ext / off / name` ต้องรอดผ่าน `cloneDay()` และ pre-place ใน `solveAll()` — เพิ่ม field ใหม่ต้องเติมทั้ง 2 จุด ไม่งั้นหายตอน regen

**slots:**

| slot | เวลา | ชม. | หมายเหตุ |
|---|---|---|---|
| `morning` | 08:00–18:00 | 10 | กะเช้า (anchor) |
| `mid` | 10:00–20:00 | 10 | คุม 2 พีค |
| `lunch` | 10:00–14:00 | 4 | พีคเที่ยง · splitOnly (เฉพาะ canSplit) |
| `dinner` | 17:00–21:00 | 4 | พีคเย็น · splitOnly |
| `night` | 17:00–03:00 | 10 | กะดึก (anchor) — เลิกตี 3 |

## 3. Engine pipeline — `solveAll(state, opts)`

เรียกผ่าน `getSol()` (memoized). best-of-8 seeds เลือกผลที่ cost ต่ำสุด:

```
cost = underTarget*8 + restViol*2 + paidLoans*0.5
     + Σ_branch( coverShort*10 + peakShort*3 + unscheduled*30 )
```

ลำดับ pass ภายใน 1 seed:

1. **pre-place** — วันไม่ regen + ช่อง `locked` วางก่อน (คง off/ghost/t/ext) · ตั้ง `mustFull` = คนที่ใช้วันหยุดไปแล้วในวันล็อก → ต้องทำครบวัน regen
2. **matchDay(d)** — bipartite matching (Kuhn's augmenting path) ต่อวัน · ไล่วันแรงก่อน `[0,1,6,2,5,3,4]` · ช่อง night→morning→mid · candidate เรียงด้วย `score()`
3. **repairCross()** — ช่องสมอขาดเพราะคนทำได้ติดเพดาน 6 วัน → ย้ายวันงานเขาให้คนว่างแทน แล้วดึงมาลงช่องขาด
4. **flexPeaks()** — เติมพีคที่ยังขาด: บรีสโมเดล (ยืดกะเช้า→08-20, ext) / อ๊อดโมเดล (คนดึก split+OT เสริมพีคเที่ยง)
5. **polish()** — สลับคู่คนวันเดียวกันถ้า score รวมดีขึ้น (คู่รถบาส/จ๋าให้กะเดียวกัน)
6. **fillTo6()** — เติมคนที่ยังไม่ครบ 6 วัน (2 รอบ: รอบแรกเว้นวันหยุดที่ขอ)
7. **restBalance()** ← **จุด focus** — Phase 1 ดันคนหยุด >1 วันไปทำงาน · Phase 2 กระจายวันหยุด ≤`maxOff`/วัน

ทุก mutation ทำผ่าน `doAssign()` / `removeAssign()` ที่ดูแล counters (`gSlots, gDays, gHours, gLoan, gTrip`) พร้อมกัน — **อย่าแก้ `weeks[][][]` ตรงๆ**

## 4. กฎทั้งหมด + จุดบังคับ

| กฎ | รายละเอียด | บังคับที่ |
|---|---|---|
| **H1** | ≤1 กะ/วัน · ยกเว้น canSplit ทำคู่เบรค (lunch+dinner) หรือ split+OT (lunch+night) | `canAddSlot`, `eligible` |
| **H3** | ≤6 วัน/สัปดาห์, ≤60 ชม. · canOt: ≤7 วัน, ≤72 ชม. | `eligible` |
| **H4** ★ | กะดึก (เลิก 03:00) → **วันถัดไปห้ามเข้า morning/mid/lunch** · เช็คสองทิศ (prevNight + next-day). **dinner/night (เริ่ม 17:00) ไม่โดนบล็อก** | `eligible`, `canServe` |
| **H5** | ต้องมีตำแหน่งใน `positions[]` | `eligible` |
| ทิศยืม | พญาไท = ส่งออกเท่านั้น (ห้ามรับ) · สาทร/สข77 = ห้ามส่งออก · ยกเว้นคู่รถ บาส+จ๋า (บ้านสาทร) สลับ สาทร↔สข77 | `canLoanTo` |
| ครบ 6 วัน | พนักงานประจำทำ 6 วัน/สัปดาห์ (off-helper ไม่นับ) | `fillTo6`, `targetOf` |
| หยุด ≤1/wk | ⚠️ แต่ละคน ≤1 วัน/สัปดาห์ (helper day = วันหยุด) | `restBalance` P1 |
| หยุด ≤N/วัน | ≤ `maxOff = max(2, ceil(staff/7))` คน/วัน/สาขา | `restBalance` P2 |
| ก่อนเริ่มงาน | วันก่อน `hired_at` ไม่จัด + ไม่นับหยุด | `hiredBlock` |

> **กุญแจ H4 ที่มักเข้าใจผิด:** กฎพักหลังกะดึกบล็อก **เฉพาะกะเริ่มเช้า** (morning/mid/lunch). คนปิดดึกตี 3 **ยังเข้ากะเย็น (dinner 17:00) หรือกะดึกวันถัดไปได้**. ใช้ช่องนี้แก้เคสอ๊อดไปแล้ว (ลง dinner ศุกร์ เป็น OT). แต่ใช้กับบีไม่ได้เพราะบีไม่ใช่ canSplit → §6

## 5. `restBalance()` — โครงที่มีอยู่

อยู่หลัง `fillTo6()`. ต่อสาขา:

```js
var maxOff = Math.max(2, Math.ceil(bEmps.length/7));  // ≤7 คน = 2
var isRest = (e,d) => !hiredBlock(e,d) && slotsOfDay(e.id,d).length===0;
// off-helper ไม่อยู่ใน gSlots → นับเป็นหยุด · ก่อนเริ่มงาน → ไม่นับ

// Phase 1: ใครหยุด >1 วัน → ดันไปทำงานวัน regen ที่หยุดอยู่
while (totalRest(e) > 1) { placeOn(e, someRestDay) || break; }

// Phase 2: วันไหนหยุดเกิน maxOff → ย้ายวันหยุดไปวันที่ยังไม่เต็ม
while (dayRest(d) > maxOff) {
  // หา rester P + วัน d2 ที่ P ทำงาน(surplus) & dayRest(d2)<maxOff
  removeAssign(P from d2); placeOn(P, d);  // P ทำวันล้น พักวันว่าง
}
```

`placeOn()` ลองเฉพาะ `night / morning / mid` (ไม่ลอง dinner/lunch เพราะ splitOnly) → นี่คือข้อจำกัดที่ทำให้ Phase 1 แก้คนดึก-ถูกบล็อกไม่ได้

## 6. ปัญหาที่ค้าง: คลาส "H4-box"

**อาการ:** คนที่คุมกะดึกหลายวัน*แบบกระจาย* (ไม่ติดกัน) มีวันหยุด >1 วัน เพราะวันถัดจากแต่ละกะดึกเข้ากะเช้าไม่ได้ (H4) และถ้าวันนั้นยังมีกะเช้า/ยืมอยู่ วันตรงกลางก็ทำอะไรไม่ได้ → กลายเป็นวันหยุดที่สอง

### เคสจริง: บี (พญาไท) สัปดาห์ 13–19 ก.ค.

```
วัน:  จ     อ      พ        พฤ    ศ     ส      อา
บี:   ดึก   หยุด   เช้า→สข   ดึก   ดึก   หยุด   เช้า→สข
```

- **อังคาร (หยุด #1):** จันทร์ดึก → อังคารเช้า/กลาง/เที่ยงบล็อก (H4) · อังคารดึก → พุธเช้า(ไปช่วยสุขุมวิท)บล็อก → อังคารทำอะไรไม่ได้
- **เสาร์ (หยุด #2):** ศุกร์ดึก → เสาร์เช้าบล็อก · เสาร์ดึก → อาทิตย์เช้า(ช่วยสุขุมวิท)บล็อก → เหมือนกัน
- บี = `positions:['kasst'], canSplit:false, canOt:false` → ลงกะเย็นสั้น (dinner) ไม่ได้ · OT ไม่ได้

### ต้นตอเชิงโครงสร้าง

**พญาไทมีผู้ช่วยครัวที่เข้าดึกได้แค่ 2 คน (บี + บีบี)** ต้องแบ่งกันคุมดึก 7 คืน → ถูกบังคับให้กระจายวัน
ตรวจแล้ว: ไม่มีคนอื่นมาแทนบีในคืนที่เขาคุมได้ (ทุกคนติดกะหรือติด H4) — ดู `tools/fix-bee.mjs`

### ทำไม post-pass แก้ไม่ได้ (ลองมาแล้ว 3 วิธี — revert หมด)

- **Phase 1b (ย้ายกะวันถัดไปให้คนอื่น):** ทำให้บี "ย้าย" วันหยุดจากวันหนึ่งไปอีกวัน — ยอดรวมเท่าเดิม (futile)
- **night-continuity score (+70 ถ้าดึกติดกัน):** fix บีได้ แต่กะดึกกระจุก → วันหยุดกองหลังบล็อกดึกพร้อมกัน → **rule 3 พัง** (หยุด 3 คน/ศุกร์)
- **restViol weight 6 ใน seed cost:** อาการเดียวกัน

→ ต้องแก้ระดับ **matching/assignment** ไม่ใช่ local repair · ปัจจุบัน seed cost มี `restViol*2` (เบา) เป็น nudge ไว้เท่านั้น

## 7. Logic ที่ต้องทำต่อ (แนะนำ 3 ทาง)

### ทาง A — Night-rotation solver ต่อสาขา ★ แนะนำ

คนคุมดึกต่อสาขามีน้อย (≤4 คน) วันมีแค่ 7 → **brute-force ได้**
เป้าหมาย: จัดกะดึกของแต่ละคนเป็น *บล็อกติดกัน* และให้บล็อกของแต่ละคน *เหลื่อมกัน* (วันหยุดกระจาย ไม่ชน rule 3)

```
// หลัง matchDay/fillTo6, ก่อน restBalance Phase 2
for each branch:
  nightStaff  = คนที่ aL && เข้าดึกช่วงสัปดาห์นี้
  nightDemand = demandOf(plan,d).night.{cook,kasst,cash}  ต่อวัน
  // แต่ละ candidate = 1 บล็อกดึกยาว L วันติดกัน (L = จำนวนคืนที่ต้องคุม)
  // เลือกชุดที่: (1) คลุม nightDemand ครบ  (2) แต่ละคนเหลือวันหยุด 1 วันติดบล็อก
  //            (3) วันหยุดรวมทุกคน ≤ maxOff/วัน  (4) ไม่ผิด H4/loan
  best = argmin over assignments of ( holes*W1 + restClusterPenalty*W2 )
  apply(best); เติมวันที่เหลือด้วยกะเช้า (fillTo6 เดิม)
```

**key insight:** บล็อกดึกติดกัน → มีวันหยุดเดียวที่ปลายบล็อก (ดึก→ดึก ถูกกฎ · ดึก→หยุด ถูกกฎ · หยุด→เช้า ถูกกฎ)
ตัวอย่างบีที่ valid (พิสูจน์แล้วไม่ผิด H4): `จ-เช้า, อ-เช้า, พ-เช้า, พฤ-ดึก, ศ-ดึก, ส-ดึก, [อา หยุด]` = 6 วัน หยุด 1 ✓

### ทาง B — Coordinated pair-swap (post-pass เบากว่า)

เจาะเฉพาะคนที่ restBalance ทิ้งไว้ >1 หยุด: หาคู่คนคุมดึกสาขาเดียวกัน (เช่น บี↔บีบี) สลับ "วันดึก" กันจนของแต่ละคนติดกัน · ต้องย้ายกะเช้า/ยืมที่เขาถืออยู่ไปคนอื่น + **ตรวจไม่ให้เกิดรู** ก่อน commit
`tools/fix-bee.mjs` เป็นโครงเริ่มต้น — ตอนนี้ bail เมื่อหาตัวแทนกะดึกไม่ได้ (ข้อจำกัดจริง)

### ทาง C — แก้ที่ข้อมูล (ops ไม่ใช่โค้ด)

ฝึกพนักงานพญาไทให้เข้าดึกได้เพิ่มอีก 1 คน (`preferred_slots += 'late'`) → คนคุมดึกมี 3 คน ไม่ต้องกระจาย
ทางถูกและถาวรสุดถ้าหน้างานทำได้ — โค้ดไม่ต้องแตะ

### Definition of Done

รัน `tools/rest-check.mjs` ทั้ง 2 สัปดาห์: **(1) หยุด ≤1/คน = ไม่มี · (3) หยุด ≤2/วัน = ไม่มี**
+ `tools/audit.mjs`: 0 กฎแตก, holes ≤ baseline โครงสร้าง (พญาไท แคชเชียร์ตรีโอ นิด/บิ๊ก/เก่ง)
+ `tools/crossweek.mjs`: ผ่าน

## 8. Verify + gotchas

### เครื่องมือตรวจ (Node ≥18, ยิง RPC ตรง — mirror กฎ engine)

| script | ตรวจ |
|---|---|
| `tools/rest-check.mjs` | 3 กฎวันหยุด (≤1/คน, ก่อนเริ่มงาน, ≤2/วัน) — **ตัวหลักของงานนี้** |
| `tools/audit.mjs` | H1/H3/H4/H5, ทิศยืม, ครบ 6 วัน, holes, OT/วันเสริม, ทริปยืม, คู่รถ |
| `tools/crossweek.mjs` | H4 ข้ามสัปดาห์ (ดึกอาทิตย์ → กะสายจันทร์สัปดาห์ถัดไป) |
| `tools/fix-bee.mjs` | โครงเริ่มต้นทาง B (`--dry` เพื่อดูก่อนบันทึก) |

สคริปต์ hardcode anon key (public — ตัวเดียวกับใน index.html, read-only ไม่ใช้ PIN) และวันที่ทดสอบ `2026-07-06 / 2026-07-13` แก้เป็น dynamic ได้ตามต้องการ

### ⚠️ ประสานงานก่อนแก้

มีงาน engine อีกตัว (**cross-week H4**) รันแยก session กำลังแก้ `solveAll()` เหมือนกัน · **merge งานนั้นก่อน** แล้วค่อยเริ่ม §7 — ทั้งสองแตะ `eligible()/canServe()` + pass ท้ายๆ ของ pipeline

### Gotchas

- **counters ต้อง sync:** ใช้ `doAssign/removeAssign` เท่านั้น (จัดการ gSlots/gDays/gHours/gLoan/gTrip ให้) · แก้ `weeks[][][]` ตรงๆ = state เพี้ยน
- **off-helper ไม่อยู่ใน gSlots:** `slotsOfDay()` ไม่เห็น → นับเป็นวันหยุด (ตั้งใจ)
- **วันล็อก (locked) = ประวัติจริง** ห้ามแก้ (จ–พฤ สัปดาห์ปัจจุบัน = ตารางที่ส่งมาจริง) · restBalance/night-solver ต้องข้ามวันล็อก
- **best-of-8 clone state ด้วย Object.assign** — field ใหม่บน state ต้อง survive (weekDates ทำไว้แล้ว)
- **hoursOf ใช้ `a.t` เป็นฐาน** ถ้ามี — ไม่ใช่ SLOTM[slot].hours · ระวังตอนคิดเพดาน ชม.

## 9. บริบทหน้างาน (domain notes)

- ชื่อเล่นที่ต้องระวัง: **"วิน" = เนวิน** (ห้ามย่อ) · **วินเทอร์ = คนละคน** (กุ๊กดึก) · **"วันศุกร์" = ชื่อคน** (ไม่ใช่วันในสัปดาห์)
- **ตารางหน้างานที่เจ้าของส่งมา = ความจริง บันทึกเป๊ะทุกชื่อทุกเวลา ห้ามตีความตัด/แปลง** (วันล็อก)
- **กะช่วยวันหยุด (off):** พนักงานมาช่วยไม่กี่ชม.ในวันหยุดตัวเอง — คิดเงิน (วันเสริมใน OT card) แต่ยังนับเป็นวันหยุด
- สาทรคนน้อย (6 คน หลังพนักงานลาออก) → ต้องพึ่งพญาไทมาช่วย · พญาไทคนเกิน = ส่งออก
- แผนกำลังคนต่อวันตั้งจากยอดขายจริง 13 สัปดาห์: จันทร์แรงสุดทุกสาขา · พฤ+ศ เบาสุด · พีคชั่วโมง 10–12 + 17–20
